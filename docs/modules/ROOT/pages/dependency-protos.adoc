= Protos from Dependencies

include::./includes/attributes.adoc[]

By default, gRPC Zero compiles `.proto` files from your project source tree only.
You can also compile protos packaged inside Maven dependency JARs and resolve imports from dependency artifacts.

== Scanning dependencies for protos to compile

Use `quarkus.generate-code.grpc.scan-for-proto` to control which dependencies are scanned for `.proto` files to generate code from.

[cols="1,3",options="header"]
|===
| Value | Behavior

| `none` (default)
| Do not scan dependencies for protos to compile.

| `all`
| Scan all runtime dependencies.

| `groupId:artifactId,...`
| Scan only the listed Maven coordinates.
|===

Example: compile protos from a specific artifact:

[source,properties]
----
quarkus.generate-code.grpc.scan-for-proto=io.example:shared-protos
----

=== Include and exclude patterns

For each dependency coordinate, you can filter which proto files are extracted:

[source,properties]
----
quarkus.generate-code.grpc.scan-for-proto=io.example:shared-protos
quarkus.generate-code.grpc.scan-for-proto-include."io.example\:shared-protos"=protobuf/**,api/v1/**
quarkus.generate-code.grpc.scan-for-proto-exclude."io.example\:shared-protos"=internal/**
----

Patterns are glob-style paths within the dependency JAR.
The coordinate in the map key must be quoted, with colons escaped as `\:`.

== Scanning dependencies for imports

Separate from compiling dependency protos, gRPC Zero resolves `import` statements by scanning dependencies for `.proto` files used as import paths.

Configure this with `quarkus.generate-code.grpc.scan-for-imports`:

[cols="1,3",options="header"]
|===
| Value | Behavior

| `com.google.protobuf:protobuf-java` (default)
| Scan only the well-known protobuf types JAR.

| `all`
| Scan all runtime dependencies for importable protos.

| `none`
| Do not scan dependencies for imports.

| `groupId:artifactId,...`
| Scan only the listed coordinates.
|===

== Sanitization of dependency protos

When extracting protos from dependencies for compilation, gRPC Zero strips `java_generic_services` options from copied files.
This avoids duplicate file errors during generation.
Generic services are deprecated in favor of classes generated by the grpc-java plugin.

A log message is emitted when this sanitization occurs.

== Test scope behavior

Dependency proto scanning for compilation is disabled during test-only codegen (`context.test()` is true).
Project protos under `src/test/proto` are still processed normally.

== Troubleshooting import errors

If the build fails while parsing a proto, check:

* The imported file exists in a scanned dependency
* `scan-for-imports` includes the dependency that packages the import
* Include/exclude patterns are not filtering out required files

See xref:troubleshooting.adoc[Troubleshooting] for parse error details.
